# 15天开发计划：多电机控制 → MoveIt → 视觉抓取

**项目目标：** 实现基于 RealSense 的自主抓取，非复杂环境下稳定运行 2 小时

**团队配置：**
- 你：底层驱动 + MoveIt 集成 + 联调（每天 4h 软件 + 4h 硬件）
- 同事（新手）：视觉模块 + 上层应用（需要学习时间）

---

## 阶段一：多电机稳定控制（Day 1-3）

### Day 1 - 单电机通信验证
**你的任务：**
- [ ] 配置 CAN 硬件（can0/can1），测试 `candump` 能看到数据
- [ ] 启动 can_driver_node，逐个测试 7 个机械臂关节
- [ ] 验证每个电机的 init/shutdown/recover 服务
- [ ] 检查位置/速度反馈是否准确（对比电机实际运动）
- [ ] 记录每个电机的响应特性（延迟、抖动）

**同事任务：**
- [ ] 安装 RealSense SDK 和 ROS 包：`sudo apt install ros-noetic-realsense2-camera`
- [ ] 运行官方 demo：`roslaunch realsense2_camera rs_camera.launch`
- [ ] 在 RViz 中查看深度图和点云
- [ ] 学习 ROS tf 基础（看官方教程）
- [ ] 阅读 MoveIt 官方文档的 "Concepts" 部分

**交付物：**
- 单电机测试报告（每个电机的通信状态、反馈精度）
- RealSense 能正常发布 `/camera/depth/image_raw` 和 `/camera/color/image_raw`

---

### Day 2 - 多电机协同测试
**你的任务：**
- [ ] 同时控制 2-3 个电机，观察 CAN 总线负载
- [ ] 测试 ros_controllers.yaml 中的控制器（JointTrajectoryController）
- [ ] 发送简单轨迹，验证多关节协同运动
- [ ] 测试限位保护（故意发送超限命令）
- [ ] 压力测试：高频命令（100Hz）、突然停止、方向突变

**同事任务：**
- [ ] 学习 ArUco 标记识别：安装 `ros-noetic-aruco-ros`
- [ ] 运行 ArUco 检测 demo，打印标记位姿
- [ ] 练习坐标变换：理解 camera_link → base_link 的 tf
- [ ] 编写简单脚本：订阅 ArUco 位姿，打印到终端
- [ ] 学习 MoveIt Python/C++ API 基础

**交付物：**
- 多电机协同运动视频
- ArUco 标记能被识别并输出位姿

---

### Day 3 - 稳定性优化
**你的任务：**
- [ ] 调整 control_frequency（测试 50Hz vs 100Hz）
- [ ] 优化 DeviceManager 的线程同步
- [ ] 测试异常场景：断线重连、电机报错、CAN 总线拥堵
- [ ] 编写监控脚本：实时显示电机状态、延迟、丢包率
- [ ] 添加日志和诊断信息（rostopic echo ~/diagnostics）

**同事任务：**
- [ ] 手眼标定准备：打印 ArUco 标记，固定在已知位置
- [ ] 学习 `easy_handeye` 包的使用
- [ ] 尝试手动标定：记录 5-10 组 (相机位姿, 机器人位姿)
- [ ] 验证标定结果：相机看到的物体位置 → 机器人坐标系
- [ ] 整理学习笔记：ROS tf、MoveIt 基础概念

**交付物：**
- 多电机连续运行 30 分钟无异常
- 监控脚本能实时显示关键指标
- 手眼标定初步结果（误差 < 5cm 即可）

---

## 阶段二：MoveIt 集成（Day 4-6）

### Day 4 - URDF 验证与 MoveIt 配置
**你的任务：**
- [ ] 检查现有 URDF：关节名称与 can_driver.yaml 一致
- [ ] 在 RViz 中加载 URDF，手动拖动关节验证模型
- [ ] 运行 MoveIt Setup Assistant：`roslaunch moveit_setup_assistant setup_assistant.launch`
- [ ] 配置运动规划组（arm group：7 个关节）
- [ ] 设置末端执行器（gripper/actuator）
- [ ] 生成 MoveIt 配置包（如 `robot_moveit_config`）

**同事任务：**
- [ ] 完成手眼标定：使用 `easy_handeye` 或手动计算
- [ ] 发布静态 tf：`camera_link` → `base_link`
- [ ] 编写测试脚本：相机识别 ArUco → 转换到 base_link → 打印坐标
- [ ] 验证精度：放置物体在已知位置，对比识别结果
- [ ] 学习 MoveIt 的 `move_group` Python 接口

**交付物：**
- MoveIt 配置包生成完成
- 手眼标定误差 < 2cm
- 能准确识别物体在机器人坐标系中的位置

---

### Day 5 - MoveIt 与 ros_control 联调
**你的任务：**
- [ ] 启动完整系统：can_driver + controller_manager + move_group
- [ ] 在 RViz MoveIt 插件中拖动目标，观察真实机械臂运动
- [ ] 测试轨迹执行：`move_group.go()` → 机械臂实际运动
- [ ] 调试同步问题：轨迹时间戳、控制器响应延迟
- [ ] 测试碰撞检测：设置障碍物，验证规划避障

**同事任务：**
- [ ] 学习 MoveIt 的 pick-place 流程
- [ ] 编写简单抓取脚本：
  - 输入：物体位姿（x, y, z, roll, pitch, yaw）
  - 输出：机械臂移动到物体上方 → 下降 → 夹取 → 抬起
- [ ] 测试固定位置抓取（手动输入坐标）
- [ ] 学习 `moveit_msgs/Grasp` 消息格式

**交付物：**
- MoveIt 能控制真实机械臂运动
- 固定坐标抓取脚本能跑通（即使没有视觉）

---

### Day 6 - 运动规划优化
**你的任务：**
- [ ] 调整 MoveIt 规划器参数（OMPL 配置）
- [ ] 测试不同规划算法（RRTConnect vs LBKPIECE）
- [ ] 优化轨迹执行速度和平滑度
- [ ] 添加安全检查：关节限位、速度限制、急停功能
- [ ] 编写启动脚本：一键启动完整系统

**同事任务：**
- [ ] 集成视觉和抓取：相机识别 ArUco → 计算抓取点 → 调用抓取脚本
- [ ] 测试完整流程（ArUco 版本）
- [ ] 添加简单的状态机：等待物体 → 识别 → 抓取 → 放置 → 循环
- [ ] 处理识别失败的情况（重试 3 次）

**交付物：**
- MoveIt 规划成功率 > 90%
- 基于 ArUco 的自动抓取能跑通（demo 级别）

## 阶段三：视觉抓取开发（Day 7-12）

## 风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 硬件故障（电机/CAN） | 中 | 高 | 准备备用电机，提前测试硬件 |
| 手眼标定精度不够 | 高 | 高 | 多次标定取平均，或用 ArUco 降级方案 |
| 新手卡住进度慢 | 高 | 中 | 提供详细模板代码，每天 code review |
| MoveIt 规划失败率高 | 中 | 中 | 调整规划器参数，简化抓取姿态 |
| 识别在复杂环境失效 | 中 | 中 | 限定使用场景，添加环境约束 |
