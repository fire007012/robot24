# can_driver 非并发类问题记录（待处理）

来源：用户提交的问题清单（2026-03-03）

## 问题列表

### P1. `onShutdown()` 后系统半死状态（生命周期）
- 现象：shutdown 后 timer/subscriber/controller 主循环仍在跑，读写链路冻结但接口仍可被调用。
- 风险：误导上层，状态与行为不一致，可恢复性差。

### P2. `write()` 比例换算存在除零/溢出风险（数值安全）
- 现象：`cmd / scale` 若 scale 为 0 或极小，可能触发 inf/越界。
- 风险：未定义行为、异常指令下发。

### P3. `joints_` 为 `std::vector`，地址稳定性设计脆弱（内存安全）
- 现象：ros_control handle 持有 `joints_` 成员地址；若未来运行期增删元素将悬空。
- 风险：后续维护中引入隐式崩溃点。

### P4. `std::stoul` 异常未捕获（错误处理）
- 现象：非法 `motor_id` 字符串可能抛异常直达上层。
- 风险：初始化失败信息不清晰，错误定位困难。

### P5. `onRecover()` 中 `motor_id == 0` 通配符语义歧义（逻辑）
- 现象：`0` 同时可能是有效 ID 与“全部电机”语义。
- 风险：无法无歧义表达目标。

### P6. 协议操作返回值缺失，失败不可见（错误处理）
- 现象：`Enable/Disable/Stop/setMode/setVelocity/setPosition` 无成功/失败反馈。
- 风险：上层持续收到“OK”，但总线或设备可能已失败。

### P7. `publishMotorStates` 精度损失和潜在窄化（数值安全）
- 现象：`double -> int32/int16` 截断；可能发生窄化问题。
- 风险：状态观测失真。

### P8. 析构阶段未先停止 ROS 通信实体（生命周期）
- 现象：析构中直接清协议/transport，timer/subscriber/service 生命周期未先收口。
- 风险：收尾阶段行为不确定。
