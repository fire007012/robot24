# 独立电机控制指南（不依赖 URDF）

本文档说明如何在**不使用 URDF** 的情况下，使用 can_driver 包独立控制单个或多个电机。

## 适用场景

- 单电机测试和调试
- 不需要机器人运动学模型的应用
- 轮式移动底盘（差速驱动、麦克纳姆轮等）
- 独立执行器控制（传送带、升降机等）
- 快速原型验证

## 核心原理

can_driver 的核心功能（CAN 通信、协议解析、电机控制）**完全独立于 URDF**。URDF 仅用于：
1. 从机器人模型读取关节限位
2. 与 MoveIt 等规划工具集成

如果不需要这些功能，可以完全跳过 URDF 配置。

## 配置步骤

### 1. 配置电机参数

编辑 `config/can_driver.yaml`：

```yaml
# 运行时参数
direct_cmd_timeout_sec: 0.5      # direct topic 命令超时时间
motor_state_period_sec: 0.1      # 状态发布周期
direct_cmd_queue_size: 1         # 命令队列大小
control_frequency: 100.0         # 控制循环频率（Hz）

# 电机配置列表
joints:
  - name: left_wheel              # 电机名称（任意，不需要与 URDF 对应）
    motor_id: 1                   # CAN 总线上的电机 ID（必须唯一）
    protocol: MT                  # 协议类型：MT 或 PP
    can_device: can0              # CAN 设备名称
    control_mode: velocity        # 控制模式：velocity 或 position
    velocity_scale: 0.01          # 速度换算系数（协议值 → rad/s）
    position_scale: 0.01          # 位置换算系数（协议值 → rad）

  - name: right_wheel
    motor_id: 2
    protocol: MT
    can_device: can0
    control_mode: velocity
    velocity_scale: 0.01
    position_scale: 0.01

  - name: lift_motor
    motor_id: 5
    protocol: PP                  # Eyou/PP 协议
    can_device: can1              # 可以使用不同的 CAN 设备
    control_mode: position
    velocity_scale: 0.001
    position_scale: 0.001
```

**重要参数说明：**
- `name`：电机名称，用于 topic/service 寻址，可以任意命名
- `motor_id`：**全局唯一**，service 命令通过此 ID 寻址电机
- `protocol`：`MT` 或 `PP`（Eyou）
- `control_mode`：`velocity`（速度控制）或 `position`（位置控制）
- `*_scale`：协议原始值到 SI 单位的换算系数

### 2. 配置关节限位（重要！）

**不使用 URDF 时，必须通过 rosparam 手动配置限位，否则命令值不会被钳制，可能损坏硬件。**

在 launch 文件中添加限位配置：

```xml
<launch>
  <!-- 加载电机配置 -->
  <rosparam command="load" file="$(find can_driver)/config/can_driver.yaml" />

  <!-- 手动配置关节限位（替代 URDF） -->
  <rosparam param="can_driver_node/joint_limits">
    left_wheel:
      has_velocity_limits: true
      max_velocity: 10.0           # 最大速度（rad/s）
      has_acceleration_limits: false
      has_position_limits: false   # 轮子通常不限制位置

    right_wheel:
      has_velocity_limits: true
      max_velocity: 10.0
      has_acceleration_limits: false
      has_position_limits: false

    lift_motor:
      has_velocity_limits: true
      max_velocity: 2.0
      has_acceleration_limits: false
      has_position_limits: true
      min_position: -1.57          # 最小位置（rad）
      max_position: 1.57           # 最大位置（rad）
  </rosparam>

  <!-- 启动驱动节点 -->
  <node name="can_driver_node" pkg="can_driver" type="can_driver_node" output="screen">
    <param name="control_frequency" value="100.0" />
  </node>
</launch>
```

**限位参数说明：**

| 参数 | 类型 | 说明 |
|------|------|------|
| `has_velocity_limits` | bool | 是否启用速度限制 |
| `max_velocity` | double | 最大速度（rad/s），速度命令会被钳制到 [-max, +max] |
| `has_position_limits` | bool | 是否启用位置限制 |
| `min_position` | double | 最小位置（rad） |
| `max_position` | double | 最大位置（rad） |
| `has_acceleration_limits` | bool | 是否启用加速度限制（当前版本未实现） |
| `max_acceleration` | double | 最大加速度（预留） |

### 3. 启动驱动

```bash
# 确保 CAN 设备已启动
sudo ip link set can0 up type can bitrate 1000000

# 启动驱动节点
roslaunch can_driver standalone_motor.launch
```

## 控制方式

### 方式 1：Direct Topic 控制（推荐用于测试）

直接发布命令到电机，绕过 ros_control 控制器。

```bash
# 速度控制
rostopic pub /can_driver_node/motor/left_wheel/cmd_velocity std_msgs/Float64 "data: 2.5"

# 位置控制
rostopic pub /can_driver_node/motor/lift_motor/cmd_position std_msgs/Float64 "data: 1.0"
```

**特点：**
- 简单直接，适合测试
- 命令会自动超时（默认 0.5 秒），超时后停止
- 优先级高于 ros_control 控制器命令

### 方式 2：Service 控制

使用 service 进行电机管理操作。

```bash
# 使能电机（motor_id=1）
rosservice call /can_driver_node/motor_command "motor_id: 1
command: 1  # CMD_ENABLE
value: 0.0"

# 失能电机
rosservice call /can_driver_node/motor_command "motor_id: 1
command: 2  # CMD_DISABLE
value: 0.0"

# 紧急停止
rosservice call /can_driver_node/motor_command "motor_id: 1
command: 3  # CMD_STOP
value: 0.0"

# 切换模式（0=位置模式，1=速度模式）
rosservice call /can_driver_node/motor_command "motor_id: 1
command: 4  # CMD_SET_MODE
value: 1.0"

# 恢复所有电机（motor_id=0xFFFF 表示全部）
rosservice call /can_driver_node/recover "motor_id: 65535"
```

**Service 命令列表：**

| 命令 | 值 | 说明 |
|------|-----|------|
| `CMD_ENABLE` | 1 | 使能电机 |
| `CMD_DISABLE` | 2 | 失能电机 |
| `CMD_STOP` | 3 | 紧急停止 |
| `CMD_SET_MODE` | 4 | 切换模式（value: 0=位置, 1=速度） |

### 方式 3：ros_control 控制器（可选）

如果需要使用标准 ros_control 控制器（如 diff_drive_controller），配置 `config/ros_controllers.yaml`：

```yaml
joint_state_controller:
  type: joint_state_controller/JointStateController
  publish_rate: 50

# 差速驱动控制器示例
diff_drive_controller:
  type: diff_drive_controller/DiffDriveController
  left_wheel: left_wheel
  right_wheel: right_wheel
  publish_rate: 50
  pose_covariance_diagonal: [0.001, 0.001, 1000000.0, 1000000.0, 1000000.0, 1000.0]
  twist_covariance_diagonal: [0.001, 0.001, 1000000.0, 1000000.0, 1000000.0, 1000.0]
  wheel_separation: 0.5
  wheel_radius: 0.1
  cmd_vel_timeout: 0.5
```

启动控制器：

```bash
roslaunch can_driver can_driver.launch load_controllers:=true
```

## 监控电机状态

### 查看电机状态

```bash
# 订阅状态 topic
rostopic echo /can_driver_node/motor_states
```

输出示例：

```yaml
motor_id: 1
name: "left_wheel"
position: 12345        # 原始协议值（未换算）
velocity: 250          # 原始协议值
current: 1500          # 电流（mA）
mode: 1                # 0=位置模式, 1=速度模式
enabled: True
fault: False
```

### 查看 joint_states（用于 RViz 可视化）

```bash
rostopic echo /joint_states
```

## 完整示例：差速驱动底盘

### 1. 配置文件

**config/wheel_motors.yaml**

```yaml
direct_cmd_timeout_sec: 0.5
motor_state_period_sec: 0.1
control_frequency: 100.0

joints:
  - name: left_wheel
    motor_id: 1
    protocol: MT
    can_device: can0
    control_mode: velocity
    velocity_scale: 0.01
    position_scale: 0.01

  - name: right_wheel
    motor_id: 2
    protocol: MT
    can_device: can0
    control_mode: velocity
    velocity_scale: 0.01
    position_scale: 0.01
```

**launch/wheel_control.launch**

```xml
<launch>
  <rosparam command="load" file="$(find can_driver)/config/wheel_motors.yaml" />

  <!-- 配置限位 -->
  <rosparam param="can_driver_node/joint_limits">
    left_wheel:
      has_velocity_limits: true
      max_velocity: 15.0
    right_wheel:
      has_velocity_limits: true
      max_velocity: 15.0
  </rosparam>

  <node name="can_driver_node" pkg="can_driver" type="can_driver_node" output="screen" />

  <!-- 可选：加载差速驱动控制器 -->
  <rosparam command="load" file="$(find can_driver)/config/diff_drive_controller.yaml" />
  <node name="controller_spawner" pkg="controller_manager" type="spawner"
        args="joint_state_controller diff_drive_controller" />
</launch>
```

### 2. 启动和测试

```bash
# 启动驱动
roslaunch can_driver wheel_control.launch

# 使能电机
rosservice call /can_driver_node/recover "motor_id: 65535"

# 方式 A：直接控制单个轮子
rostopic pub /can_driver_node/motor/left_wheel/cmd_velocity std_msgs/Float64 "data: 5.0"
rostopic pub /can_driver_node/motor/right_wheel/cmd_velocity std_msgs/Float64 "data: 5.0"

# 方式 B：使用差速驱动控制器（需要配置 ros_controllers.yaml）
rostopic pub /diff_drive_controller/cmd_vel geometry_msgs/Twist "linear: {x: 0.5}
angular: {z: 0.0}"
```

## 故障排查

### 1. 电机不响应

**检查 CAN 设备：**

```bash
# 查看 CAN 设备状态
ip link show can0

# 查看 CAN 流量
candump can0
```

**检查驱动日志：**

```bash
# 查看是否有初始化错误
rosnode info /can_driver_node
rostopic echo /rosout | grep can_driver
```

### 2. 命令被钳制

**现象：** 发送大命令值，但电机只执行到某个上限。

**原因：** 关节限位生效。

**解决：** 检查并调整限位配置：

```bash
# 查看当前限位配置
rosparam get /can_driver_node/joint_limits
```

### 3. 没有配置限位的警告

**日志输出：**

```
[WARN] [CanDriverHW] Joint 'left_wheel': no limits found, commands will not be clamped.
```

**解决：** 在 launch 文件中添加 `joint_limits` 配置（见上文）。

### 4. motor_id 冲突

**错误：**

```
[ERROR] [CanDriverHW] Duplicate motor_id=1 in joints config.
```

**原因：** 多个 joint 配置了相同的 `motor_id`。

**解决：** 确保每个电机的 `motor_id` 全局唯一。

## 与 URDF 模式的对比

| 特性 | 独立模式（无 URDF） | URDF 模式 |
|------|---------------------|-----------|
| 配置复杂度 | 低（只需 YAML） | 中（需要 URDF + YAML） |
| 关节限位 | 手动配置 rosparam | 自动从 URDF 读取 |
| MoveIt 集成 | 不支持 | 支持 |
| RViz 可视化 | 仅 joint_states | 完整机器人模型 |
| 适用场景 | 单电机测试、轮式底盘 | 机械臂、复杂机器人 |
| 启动速度 | 快 | 慢（需加载 URDF） |

---

## 最佳实践

1. **始终配置关节限位**，即使是测试环境，避免硬件损坏
2. **使用 vcan 进行离线测试**：
   ```bash
   sudo modprobe vcan
   sudo ip link add dev vcan0 type vcan
   sudo ip link set up vcan0
   ```
3. **先用 service 使能电机**，再发送运动命令
4. **监控 motor_states**，确认电机响应正常
5. **velocity_scale 和 position_scale** 根据实际电机编码器分辨率调整
