# 开发步骤

## 概述

本文档给出从当前状态到系统完整可运行的全部开发步骤。
核心原则：**两个驱动包的代码都不修改**，只做配置调整和新增代理节点。

---

## 阶段一：统一关节命名与 URDF

### 目标
确保全系统 12 个关节在 URDF、两个驱动包的参数文件、controller 配置中使用完全一致的名称。

### 步骤

1. **确定关节命名表**

   | 关节名 | 子系统 | 驱动节点 |
   |--------|--------|----------|
   | arm_joint_1 ~ arm_joint_4 | 机械臂 | can_driver_node |
   | arm_joint_5 ~ arm_joint_6 | 机械臂 | canopen_motor_node |
   | left_track, right_track | 底盘 | can_driver_node |
   | flipper_1 ~ flipper_4 | 摆臂 | canopen_motor_node |

2. **更新 URDF**
   - 确认 URDF 中 12 个 joint 的 `name` 属性与上表一致
   - 确认每个 joint 的 `type`（revolute / continuous）正确
   - 填写关节限位（position / velocity / effort limits）

3. **更新 can_driver.yaml**
   - `joints` 列表中 6 个关节的 `name` 字段改为上表名称
   - 确认 arm_joint_1~4 的 `control_mode: position`
   - 确认 left_track / right_track 的 `control_mode: velocity`

4. **更新 yiyou_canopen 的 motor.yaml**
   - 6 个关节的 joint name 改为上表名称
   - 确认 arm_joint_5~6 和 flipper_1~4 都是位置模式

5. **验证**
   ```bash
   # 检查 URDF 解析
   check_urdf robot.urdf

   # 分别启动两个驱动节点，查看 joint_states 的 name 字段
   rostopic echo /can_driver/joint_states/name
   rostopic echo /canopen/joint_states/name
   ```

## 阶段二：配置 Controller

### 目标
在两个驱动节点中分别加载正确的 controller，使每个子系统都有对应的控制器。

### 步骤

1. **can_driver_node 的 ros_controllers.yaml**

   ```yaml
   joint_state_controller:
     type: joint_state_controller/JointStateController
     publish_rate: 100

   pp_arm_controller:
     type: position_controllers/JointTrajectoryController
     joints:
       - arm_joint_1
       - arm_joint_2
       - arm_joint_3
       - arm_joint_4
     constraints:
       goal_time: 5.0
       arm_joint_1: {goal: 0.02}
       arm_joint_2: {goal: 0.02}
       arm_joint_3: {goal: 0.02}
       arm_joint_4: {goal: 0.02}

   diff_drive_controller:
     type: diff_drive_controller/DiffDriveController
     left_wheel: left_track
     right_wheel: right_track
     publish_rate: 50
     wheel_separation: 0.5     # 替换为实际轮距
     wheel_radius: 0.1         # 替换为实际轮径
   ```

2. **canopen_motor_node 的 controllers 配置**

   ```yaml
   joint_state_controller:
     type: joint_state_controller/JointStateController
     publish_rate: 100

   canopen_arm_controller:
     type: position_controllers/JointTrajectoryController
     joints:
       - arm_joint_5
       - arm_joint_6
     constraints:
       goal_time: 5.0
       arm_joint_5: {goal: 0.02}
       arm_joint_6: {goal: 0.02}

   flipper_controller:
     type: position_controllers/JointGroupPositionController
     joints:
       - flipper_1
       - flipper_2
       - flipper_3
       - flipper_4
   ```

3. **更新 launch 文件**
   - can_driver.launch 中添加 controller_spawner：
     ```xml
     <node name="controller_spawner" pkg="controller_manager" type="spawner"
           args="joint_state_controller pp_arm_controller diff_drive_controller"/>
     ```
   - yiyou_canopen.launch 中添加 controller_spawner：
     ```xml
     <node name="controller_spawner" pkg="controller_manager" type="spawner"
           args="joint_state_controller canopen_arm_controller flipper_controller"/>
     ```

4. **验证**
   ```bash
   # 启动 can_driver_node
   roslaunch can_driver can_driver.launch

   # 检查 controller 是否加载成功
   rosservice call /can_driver_node/controller_manager/list_controllers

   # 检查 action server 是否存在
   rostopic list | grep follow_joint_trajectory

   # 同样验证 canopen_motor_node
   roslaunch yiyou_canopen yiyou_canopen.launch
   rosservice call /canopen/controller_manager/list_controllers
   ```

## 阶段三：joint_states 合并

### 目标
把两个驱动节点各自发布的 joint_states 合并为统一的 `/joint_states`，供 MoveIt 和 robot_state_publisher 使用。

### 步骤

1. **在各自 launch 中 remap joint_states**

   can_driver.launch：
   ```xml
   <node name="can_driver_node" pkg="can_driver" type="can_driver_node">
     <remap from="joint_states" to="can_driver/joint_states"/>
     <!-- 其他参数 -->
   </node>
   ```

   yiyou_canopen.launch：
   ```xml
   <!-- canopen_motor_node 发布的 joint_states remap -->
   <remap from="joint_states" to="canopen/joint_states"/>
   ```

2. **添加 joint_state_publisher 合并节点**

   在顶层 launch 中：
   ```xml
   <node name="joint_state_publisher" pkg="joint_state_publisher"
         type="joint_state_publisher">
     <rosparam param="source_list">
       - can_driver/joint_states
       - canopen/joint_states
     </rosparam>
   </node>
   ```

3. **添加 robot_state_publisher**
   ```xml
   <node name="robot_state_publisher" pkg="robot_state_publisher"
         type="robot_state_publisher"/>
   ```

4. **验证**
   ```bash
   # 检查合并后的 joint_states
   rostopic echo /joint_states

   # 应该能看到 12 个关节的名称和数据
   # 检查 TF 树
   rosrun tf view_frames
   ```

## 阶段四：编写 arm_traj_splitter 代理节点

### 目标
实现 C++ 代理节点，对 MoveIt 暴露 6 轴 FollowJointTrajectory action，
内部拆分为 4+2 分发给两个后端。

### 步骤

1. **创建包**
   ```bash
   cd ~/Robot24_catkin_ws/src
   catkin_create_pkg arm_traj_splitter roscpp actionlib control_msgs trajectory_msgs
   ```

2. **实现核心代码**

   按 `05_arm_traj_splitter设计.md` 中的设计实现：
   - `SplitterNode.h`：类声明
   - `SplitterNode.cpp`：onGoal / splitTrajectory / splitTolerances
   - `main.cpp`：节点入口

   关键实现要点：
   - 关节归属从参数文件加载（`splitter.yaml`），不硬编码
   - `splitTrajectory` 中按 joint_names 建立索引映射，拆分 positions / velocities / accelerations
   - `time_from_start` 保持不变，保证两个后端的时间基准一致
   - 轮询等待时检查 preempt 请求，任一方失败立即取消另一方

3. **配置文件**

   `config/splitter.yaml`：
   ```yaml
   arm_controller_name: "arm_controller"
   backend_controllers:
     - name: "pp_arm_controller"
       action_ns: "/can_driver_node/pp_arm_controller/follow_joint_trajectory"
       joints:
         - arm_joint_1
         - arm_joint_2
         - arm_joint_3
         - arm_joint_4
     - name: "canopen_arm_controller"
       action_ns: "/canopen/canopen_arm_controller/follow_joint_trajectory"
       joints:
         - arm_joint_5
         - arm_joint_6
   ```

4. **Launch 文件**

   `launch/splitter.launch`：
   ```xml
   <launch>
     <node name="arm_traj_splitter" pkg="arm_traj_splitter"
           type="arm_traj_splitter_node" output="screen">
       <rosparam file="$(find arm_traj_splitter)/config/splitter.yaml"
                 command="load"/>
     </node>
   </launch>
   ```

5. **编译与测试**
   ```bash
   cd ~/Robot24_catkin_ws
   catkin_make

   # 启动全部节点后，用命令行测试 action
   rostopic list | grep arm_controller
   # 应该看到 /arm_controller/follow_joint_trajectory/*

   # 用 actionlib 命令行工具发送测试轨迹
   rosrun actionlib_tools axclient.py /arm_controller/follow_joint_trajectory
   ```

## 阶段五：MoveIt 配置

### 目标
配置 MoveIt，使其通过代理节点控制 6 轴机械臂。

### 步骤

1. **MoveIt Setup Assistant 生成配置包**（如尚未生成）
   ```bash
   roslaunch moveit_setup_assistant setup_assistant.launch
   ```
   - 加载 URDF
   - 定义 planning group `arm`，包含 arm_joint_1 ~ arm_joint_6
   - 生成配置包（如 `robot_moveit_config`）

2. **修改 controllers.yaml**

   `robot_moveit_config/config/controllers.yaml`：
   ```yaml
   controller_list:
     - name: arm_controller
       action_ns: follow_joint_trajectory
       type: FollowJointTrajectory
       joints:
         - arm_joint_1
         - arm_joint_2
         - arm_joint_3
         - arm_joint_4
         - arm_joint_5
         - arm_joint_6
   ```

3. **修改 joint_limits.yaml**（如有特殊限位需求）

4. **验证**
   ```bash
   # 启动全部系统
   roslaunch robot_bringup full_system.launch

   # 启动 MoveIt
   roslaunch robot_moveit_config move_group.launch

   # 启动 RViz
   roslaunch robot_moveit_config moveit_rviz.launch

   # 在 RViz 中拖拽目标位置，Plan & Execute
   ```

## 阶段六：底盘与摆臂接入

### 目标
完成底盘差速控制和摆臂遥控接入。

### 步骤

1. **底盘 diff_drive_controller 参数调优**
   - 测量实际轮距（`wheel_separation`）
   - 测量实际驱动轮/履带等效半径（`wheel_radius`）
   - 在 ros_controllers.yaml 中填入正确值
   - 注意：履带底盘的差速模型可能需要额外的滑移补偿参数

2. **验证底盘**
   ```bash
   # 发送速度命令
   rostopic pub /cmd_vel geometry_msgs/Twist \
     "linear: {x: 0.1}" -r 10

   # 观察底盘是否运动
   # 检查里程计
   rostopic echo /diff_drive_controller/odom
   ```

3. **摆臂遥控映射**
   - 如果使用 `joy_node`，写一个小节点把摇杆值映射到 4 个摆臂角度
   - 发布到 `/flipper_controller/command`（Float64MultiArray）

4. **验证摆臂**
   ```bash
   # 手动发送摆臂命令
   rostopic pub /flipper_controller/command std_msgs/Float64MultiArray \
     "data: [0.1, 0.1, -0.1, -0.1]"
   ```

## 阶段七：顶层 Launch 整合

### 目标
一个 launch 文件启动完整系统。

### 步骤

1. **创建顶层 bringup 包**
   ```bash
   catkin_create_pkg robot_bringup roslaunch
   ```

2. **编写 full_system.launch**
   ```xml
   <launch>
     <!-- ===== URDF ===== -->
     <param name="robot_description"
            command="$(find xacro)/xacro '$(find robot_description)/urdf/robot.urdf.xacro'"/>

     <!-- ===== 驱动层 ===== -->
     <!-- 私有协议电机 (can_B) -->
     <include file="$(find can_driver)/launch/can_driver.launch">
       <arg name="can_device" value="can1"/>
     </include>

     <!-- CANopen 电机 (can_A) -->
     <include file="$(find yiyou_canopen)/launch/yiyou_canopen.launch">
       <arg name="can_device" value="can0"/>
     </include>

     <!-- ===== 状态合并 ===== -->
     <node name="joint_state_publisher" pkg="joint_state_publisher"
           type="joint_state_publisher">
       <rosparam param="source_list">
         - can_driver/joint_states
         - canopen/joint_states
       </rosparam>
     </node>

     <node name="robot_state_publisher" pkg="robot_state_publisher"
           type="robot_state_publisher"/>

     <!-- ===== 机械臂轨迹代理 ===== -->
     <include file="$(find arm_traj_splitter)/launch/splitter.launch"/>

     <!-- ===== MoveIt ===== -->
     <include file="$(find robot_moveit_config)/launch/move_group.launch"/>
   </launch>
   ```

3. **启动前检查清单**
   ```bash
   # CAN 接口就绪
   sudo ip link set can0 up type can bitrate 1000000
   sudo ip link set can1 up type can bitrate 1000000

   # 检查 CAN 接口状态
   ip -details link show can0
   ip -details link show can1

   # 一键启动
   roslaunch robot_bringup full_system.launch
   ```

## 开发阶段总览

| 阶段 | 内容 | 预计工作量 | 依赖 |
|------|------|-----------|------|
| 一 | 统一关节命名 + URDF | 半天 | 无 |
| 二 | 配置 Controller | 半天 | 阶段一 |
| 三 | joint_states 合并 | 2 小时 | 阶段二 |
| 四 | arm_traj_splitter 代理节点 | 1~2 天 | 阶段二 |
| 五 | MoveIt 配置 | 半天 | 阶段三 + 四 |
| 六 | 底盘 + 摆臂接入 | 半天 | 阶段二 |
| 七 | 顶层 Launch 整合 | 2 小时 | 全部 |

**总预计工作量：3~4 天**

阶段一 ~ 三可并行准备，阶段四是主要开发工作，
阶段五 ~ 七在阶段四完成后快速收尾。
