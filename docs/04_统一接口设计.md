## 1. 设计原则

上层应用（MoveIt、Navigation、遥控器）只与标准化接口交互，
不感知底层是 `ros_canopen` 还是 `can_driver`，也不感知有几条 CAN 线。

## 2. 对外暴露的四个接口

| 接口 | 类型 | 方向 | 用途 | 使用者 |
|------|------|------|------|--------|
| `/arm_controller/follow_joint_trajectory` | FollowJointTrajectory action | 写 | 6 轴机械臂轨迹控制 | MoveIt |
| `/cmd_vel` | `geometry_msgs/Twist` topic | 写 | 底盘差速驱动 | Navigation / 遥控 |
| `/flipper_controller/command` | `std_msgs/Float64MultiArray` topic | 写 | 4 摆臂位置控制 | 遥控器 |
| `/joint_states` | `sensor_msgs/JointState` topic | 读 | 全部 12 关节状态 | MoveIt / RViz / TF |

## 3. 关节命名约定

全系统 12 个关节使用统一命名。URDF、驱动配置、controller 配置、MoveIt 配置必须一致。

| 关节名 | 子系统 | 驱动节点 |
|--------|--------|----------|
| `arm_joint_1` | 机械臂 | `can_driver_node` |
| `arm_joint_2` | 机械臂 | `can_driver_node` |
| `arm_joint_3` | 机械臂 | `can_driver_node` |
| `arm_joint_4` | 机械臂 | `can_driver_node` |
| `arm_joint_5` | 机械臂 | `canopen_motor_node` |
| `arm_joint_6` | 机械臂 | `canopen_motor_node` |
| `left_track` | 底盘 | `can_driver_node` |
| `right_track` | 底盘 | `can_driver_node` |
| `flipper_1` | 摆臂 | `canopen_motor_node` |
| `flipper_2` | 摆臂 | `canopen_motor_node` |
| `flipper_3` | 摆臂 | `canopen_motor_node` |
| `flipper_4` | 摆臂 | `canopen_motor_node` |

## 4. 必须对齐的五个契约项

| 契约项 | 说明 | 落地方式 |
|--------|------|----------|
| 关节命名 | 两个包发布/接收的 joint name 完全相同 | 统一 yaml + URDF |
| 单位 | 北向一律 rad / rad/s | 两边都做 counts↔rad 换算 |
| Controller 配置 | 共享一致的 controller 定义 | 对齐 `controllers.yaml` |
| Action / Topic 名 | MoveIt 配置里固定 action 名 | launch remap 对齐 |
| 关节限位 | 同一份 URDF 与限位参数 | URDF + `joint_limits.yaml` |

## 6. joint_states 合并

两个驱动节点各自发布部分关节状态，需要合并为统一的 `/joint_states`。

### can_driver_node 侧

remap 必须写在 `<node>` 标签内部才能生效：

```xml
<node name="can_driver_node" pkg="can_driver" type="can_driver_node"
      output="screen">
  <!-- remap 写在 node 作用域内，只影响该节点 -->
  <remap from="joint_states" to="can_driver/joint_states"/>
  <rosparam file="$(find can_driver)/config/can_driver.yaml" command="load"/>
  <rosparam file="$(find can_driver)/config/ros_controllers.yaml" command="load"/>
</node>
```

### canopen_motor_node 侧

ros_canopen 的 `canopen_motor_node` 默认把 joint_states 发布在自己的命名空间下
（例如 `/canopen/joint_states`），具体取决于 launch 中的 `ns` 设定。
如果需要显式 remap，同样必须在 `<node>` 或包裹的 `<group>` 内：

```xml
<group ns="canopen">
  <node name="canopen_motor_node" pkg="canopen_motor_node"
        type="canopen_motor_node" output="screen">
    <!-- 如果节点默认发布到 joint_states，remap 到带前缀的 topic -->
    <remap from="joint_states" to="/canopen/joint_states"/>
    <!-- 其他参数加载 -->
  </node>

  <!-- controllers yaml 加载到 canopen 命名空间下（见补丁 4） -->
  <rosparam file="$(find yiyou_canopen)/config/controllers.yaml" command="load"/>

  <node name="controller_spawner" pkg="controller_manager" type="spawner"
        args="joint_state_controller canopen_arm_controller flipper_controller"/>
</group>
```

> **注意**：`<remap>` 写在 `<launch>` 顶层或 `<include>` 外部时，
> 只对同级 `<node>` 生效，不会穿透到被 include 的 launch 内部。
> 务必确认 remap 在目标节点的作用域内。

### 合并节点

```xml
<node name="joint_state_publisher" pkg="joint_state_publisher"
      type="joint_state_publisher">
  <rosparam param="source_list">
    - can_driver/joint_states
    - canopen/joint_states
  </rosparam>
</node>

<node name="robot_state_publisher" pkg="robot_state_publisher"
      type="robot_state_publisher"/>
```

## 7. MoveIt 配置约定

MoveIt 的 `controllers.yaml` 指向统一 Action：

```yaml
controller_list:
  - name: arm_controller
    action_ns: follow_joint_trajectory
    type: FollowJointTrajectory
    joints:
      - arm_joint_1
      - arm_joint_2
      - arm_joint_3
      - arm_joint_4
      - arm_joint_5
      - arm_joint_6
```

MoveIt 只看到 `/arm_controller/follow_joint_trajectory`，
后端由代理节点拆分到两个驱动。
