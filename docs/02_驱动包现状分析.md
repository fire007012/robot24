# 02 驱动包现状分析

## 文档目标
- 对比现有驱动包的职责边界、接口形式与适用场景。

## 包职责（当前）
- `can_driver`：自研 `RobotHW` 驱动，支持 MT/PP 协议混合。
- `can_driver_ui`：调试与桥接，服务于联调与手动控制。
- `yiyou_canopen`：`ros_canopen` 配置包，面向 CiA402 伺服接入。

## 差异点
- 控制入口不完全一致（direct topic/service 与 canopen 风格接口并存）。
- MoveIt 集成路径存在差异（控制器配置与启动方式不同）。
