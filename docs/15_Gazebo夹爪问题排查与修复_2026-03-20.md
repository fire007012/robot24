# Gazebo 夹爪问题排查与修复（2026-03-20）

## 1. 问题现象

本次联调中，夹爪在 Gazebo + MoveIt + RViz 下出现以下问题：

1. 夹爪“松胯”、不跟随目标，存在失控感。
2. RViz 里设置夹爪目标后会“立刻回退”到初始附近。
3. 在某些阶段，夹爪位置看似被卡在约 `0.010`（正向）和 `-0.001`（负向）附近。
4. 使用 `rostopic pub /gripper_controller/command ...` 直接发指令时，Gazebo 端可动作，说明部分控制链路正常。

## 2. 关键排查结论

### 2.1 mimic 插件依赖与库名不匹配

- URDF 最初使用：
  - `libroboticsgroup_gazebo_mimic_joint_plugin.so`
- 系统实际安装包与库文件为：
  - 包：`roboticsgroup_upatras_gazebo_plugins`
  - 库：`libroboticsgroup_upatras_gazebo_mimic_joint_plugin.so`

结果：右爪 mimic 联动异常，表现为夹爪状态不稳定。

### 2.2 MoveIt 夹爪组与执行控制器关节不一致

- SRDF 的 `gripper` 组曾包含左右两关节。
- 实际 `gripper_controller` 只驱动 `left_gripper_finger_joint`。

结果：MoveIt 规划/执行与可控关节集合不一致，导致在 RViz 中出现目标回退现象。

### 2.3 夹爪动力学与 PID 初值偏软/不匹配

通过状态对比可见：
- `desired` 已到 `-0.0219` 附近
- `actual` 长时间停留在 `-0.002` 附近

这说明 MoveIt 目标已发出且 action 返回成功，但 Gazebo 内关节无法有效跟踪目标（推力不足或阻力过大）。

## 3. 已实施修复

### 3.1 URDF 夹爪关节参数

文件：`car_urdf/urdf/car_urdf.urdf`

1. 夹爪关节限位与能力：
   - `effort: 20`
   - `velocity: 0.2`
2. 添加/调整关节动力学：
   - `damping: 0.2`
   - `friction: 0.02`

### 3.2 mimic 插件修正

文件：`car_urdf/urdf/car_urdf.urdf`

1. 插件库名改为实际安装库：
   - `libroboticsgroup_upatras_gazebo_mimic_joint_plugin.so`
2. `maxEffort` 提升至 `20.0`。
3. 夹爪控制链路下，`robotNamespace` 使用根命名空间（`/`）保证当前启动链路可发现。

### 3.3 MoveIt 语义与规划配置一致化

文件：`car_moveit_config/config/car_urdf.srdf`

1. `gripper` 组仅保留：
   - `left_gripper_finger_joint`
2. 将 `right_gripper_finger_joint` 标记为 `passive_joint`（由 mimic 跟随）。

文件：`car_moveit_config/config/ompl_planning.yaml`

1. `gripper` 的 `projection_evaluator` 改为单关节：
   - `joints(left_gripper_finger_joint)`

### 3.4 MoveIt 关节限位显式化

文件：`car_moveit_config/config/joint_limits.yaml`

为夹爪显式设置：
- `has_position_limits: true`
- `min_position: -0.022`
- `max_position: 0.03`

避免运行时参数继承/覆盖导致的隐式夹限。

### 3.5 轨迹执行参数

文件：`car_moveit_config/launch/trajectory_execution.launch.xml`

- `trajectory_execution/allowed_start_tolerance`：`0.01 -> 0.05`

用于降低起点检查过严导致的回退风险。

### 3.6 Gazebo PID 增益

文件：`car_moveit_config/config/gazebo_controllers.yaml`

夹爪 PID 调整为：
- `left_gripper_finger_joint: {p: 1200.0, i: 2.0, d: 20.0, i_clamp: 1.0}`
- `right_gripper_finger_joint: {p: 1200.0, i: 2.0, d: 20.0, i_clamp: 1.0}`

## 4. 关键验证结果

1. `FollowJointTrajectory` 返回 `status: 3, error_code: 0`。
2. MoveIt 发出的轨迹目标点可达约 `0.0299`，不再被限制在 `0.010/-0.001` 附近。
3. 经调整后夹爪突破原有卡滞区间，可按目标位置运动。

## 5. 推荐验收命令（回归测试）

```bash
# 1) 控制器状态
rosservice call /controller_manager/list_controllers

# 2) MoveIt 关节限位参数
rosparam get /robot_description_planning/joint_limits/left_gripper_finger_joint
rosparam get /robot_description_planning/joint_limits/right_gripper_finger_joint

# 3) 夹爪 action 结果
rostopic echo /gripper_controller/follow_joint_trajectory/result

# 4) 夹爪控制器跟踪状态（关注 desired/actual/error）
rostopic echo /gripper_controller/state
```

## 6. 经验与防回归建议

1. 对“mimic 插件”务必校验库名与系统安装包一致，不要混用旧命名。
2. MoveIt 的 group joints 必须与可执行控制器 joints 一致；mimic 从动关节应在 SRDF 中标记为 passive。
3. 仅看 action 成功不足以判断物理执行成功，需同时检查 `/gripper_controller/state` 的 `desired/actual/error`。
4. 调参优先顺序建议：
   - 先修配置一致性（URDF/SRDF/controller）
   - 再修插件依赖
   - 最后做动力学与 PID 收敛
