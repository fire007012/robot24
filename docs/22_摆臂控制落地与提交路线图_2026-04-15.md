# 摆臂控制落地与提交路线图（2026-04-15）

## 1. 文档目的

本文用于把当前四摆臂控制改造拆成一组清晰、可审阅、可回滚的提交序列，避免把 CANopen 配置、统一执行入口配置、上层控制包实现和设计文档混成一个大提交。

---

## 2. 当前已落地范围

当前工作区已经完成以下内容：

- 新建 `flipper_control` 包
- 实现 `flipper_manager_node`
- 新增 `FlipperControlState`、`SetControlProfile`、`SetLinkageMode`
- 打通 `csp_position`、`csp_jog`、`csv_velocity`
- 在 `Eyou_ROS1_Master` 中补齐摆臂控制器配置与默认启动集

- 更新顶层文档，明确生命周期 authority、摆臂模式分层设计和当前工程边界

## 4. 如果想进一步压缩提交数

若你希望提交数更少，推荐压缩成以下 3 个提交：

1. `docs + CANopen config`
2. `Eyou_ROS1_Master controller wiring`
3. `flipper_control package + tests`

不建议压成单个提交，因为：

- `Eyou_Canopen_Master` 是相对独立的仓库边界
- DCF 生成文件会放大 diff
- 上层包实现和底层主站配置混在一起后，不利于回滚与审阅
