# 迁移设备外部依赖清单

本文档面向当前 `Robot24_catkin_ws/src` 工作空间的 ROS1 主链路迁移安装。

目标：

- 列出当前正式链路需要系统安装的外部依赖
- 区分“当前主链路必需”和“视觉/测试/示例可选”
- 避免把仓库内自带包、归档包、ROS2 示例包误当成外部依赖

## 1. 判定范围

当前正式 ROS1 主链路为：

- `robot_bringup`
- `Eyou_ROS1_Master`
- `Eyou_Canopen_Master`
- `can_driver`
- `mobility_control`
- `flipper_control`
- `arm_control`
- `car_moveit_config`
- `car_urdf`

已排除出“迁移设备必装”范围的内容：

- `control/`
  说明：归档参考包，不参与正式 bringup
- `can_driver/lib/myactuator_rmd_ros-main/`
  说明：ROS2/ament 示例，不属于当前 ROS1 主链路
- `vision_pkg`、`realsense-ros`、`vision_opencv`
  说明：视觉链路按需安装

## 2. 推荐安装方式

优先使用 `rosdep`：

```bash
cd ~/Robot24_catkin_ws
source /opt/ros/noetic/setup.bash
rosdep install --from-paths src --ignore-src -r -y --rosdistro noetic
```

但当前仓库存在两类情况，单靠 `rosdep` 不够：

- 有些 CMake 级别依赖没有完整写进 `package.xml`
- 仓库里混有 ROS2 示例包，会把 `gazebo_ros2_control`、`ros2_control` 之类键也带出来

因此迁移机器建议按下面的人工清单兜底安装。

## 3. ROS1 主链路必需依赖

```bash
sudo apt-get install -y \
  ros-noetic-roscpp \
  ros-noetic-rospy \
  ros-noetic-std-msgs \
  ros-noetic-std-srvs \
  ros-noetic-sensor-msgs \
  ros-noetic-geometry-msgs \
  ros-noetic-nav-msgs \
  ros-noetic-trajectory-msgs \
  ros-noetic-actionlib \
  ros-noetic-actionlib-msgs \
  ros-noetic-control-msgs \
  ros-noetic-diagnostic-msgs \
  ros-noetic-diagnostic-updater \
  ros-noetic-message-generation \
  ros-noetic-message-runtime
```

### 3.2 ros_control 与控制器

```bash
sudo apt-get install -y \
  ros-noetic-controller-manager \
  ros-noetic-controller-manager-msgs \
  ros-noetic-hardware-interface \
  ros-noetic-joint-limits-interface \
  ros-noetic-joint-state-controller \
  ros-noetic-diff-drive-controller \
  ros-noetic-position-controllers \
  ros-noetic-effort-controllers \
  ros-noetic-velocity-controllers
```

说明：

- `velocity_controllers` 现在 `rospack` 曾报缺失，迁机时建议明确安装
- 即使当前主链路主要用 `joint_trajectory_controller`/`diff_drive_controller`，控制器族建议一并装齐，省得后续联调时再补

### 3.3 机器人模型、TF、启动与可视化

### 3.4 MoveIt

### 3.5 Gazebo 仿真

```bash
sudo apt-get install -y \
  gazebo11 \
  ros-noetic-gazebo-msgs
```

说明：

- 当前正式 ROS1 仿真链路使用 Gazebo
- `gazebo_ros2_control` 属于仓库中 ROS2 示例依赖，不是当前 ROS1 主链路必装项

### 3.6 常规工具与系统库

```bash
sudo apt-get install -y \
  build-essential \
  cmake \
  pkg-config \
  libboost-all-dev \
  libeigen3-dev \
  libyaml-cpp-dev \
  libspdlog-dev \
  python3-yaml \
  python3-numpy \
  python3-opencv
```

### 3.7 CANopen / Lely 依赖

`Eyou_Canopen_Master` 与 `Eyou_ROS1_Master` 的 `CMakeLists.txt` 明确使用了：

- `liblely-coapp`
- `liblely-io2`
- `liblely-ev`
- `liblely-co`

也就是需要安装 Lely Core Libraries 对应开发包。

这部分在不同系统源上的包名可能不完全一致，当前仓库是通过 `pkg-config` 查找：

```cmake
pkg_check_modules(LELY REQUIRED liblely-coapp liblely-io2 liblely-ev liblely-co)
```

迁移时至少要满足：

```bash
pkg-config --modversion liblely-coapp
pkg-config --modversion liblely-io2
pkg-config --modversion liblely-ev
pkg-config --modversion liblely-co
```

如果目标机器的软件源没有现成包，需要单独安装/编译 Lely。

## 4. 视觉链路可选依赖

如果迁移设备还需要视觉功能，再额外安装下面这些。

### 4.1 ROS 视觉包

```bash
sudo apt-get install -y \
  ros-noetic-image-transport \
  ros-noetic-nodelet \
  ros-noetic-ddynamic-reconfigure \
  ros-noetic-joy
```

### 4.2 OpenCV / cv_bridge

当前仓库自带：

- `vision_opencv/cv_bridge`
- `vision_opencv/image_geometry`

同时 `vision_pkg` 还带了自有 `third_party/opencv4100lib`。

因此：

- 如果只跑当前主链路，`libopencv-dev` 不是硬性前置
- 如果要编译视觉链路，建议仍安装系统 OpenCV 与 Python OpenCV：

```bash
sudo apt-get install -y \
  libopencv-dev \
  python3-opencv \
  python3-numpy
```

### 4.3 RealSense

如果需要 `realsense2_camera`：

```bash
sudo apt-get install -y \
  ros-noetic-realsense2-camera \
  ros-noetic-realsense2-description
```

如果走系统 SDK 方式，再补：

```bash
sudo apt-get install -y \
  librealsense2-dev \
  librealsense2-dkms
```

说明：

- `rosdep resolve librealsense2` 在当前环境里返回的是 `ros-noetic-librealsense2`
- 但实际设备迁移通常还会直接需要 `librealsense2-dev` 和内核 DKMS 规则

## 5. 开发 / 测试可选依赖

## 7. 迁移后检查命令

```bash
source /opt/ros/noetic/setup.bash
cd ~/Robot24_catkin_ws
rosdep check --from-paths src --ignore-src -r --rosdistro noetic
catkin_make
```

如果只检查正式入口：

```bash
roslaunch robot_bringup full_system.launch enable_rviz:=false
roslaunch robot_bringup full_system_simulate.launch enable_rviz:=false gazebo_gui:=false
```

## 8. 备注

- 本清单按 2026-04-19 当前仓库状态整理
- `package.xml` 能覆盖大部分 ROS 依赖，但不能完全覆盖原生库依赖，因此额外补了 `yaml-cpp`、`spdlog`、Lely 等 CMake 级依赖
- 仓库中的 ROS2 示例包没有纳入当前 ROS1 主链路迁移范围
