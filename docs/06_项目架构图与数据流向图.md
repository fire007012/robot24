## 1. 总体架构图

```text
┌═══════════════════════════════════════════════════════════════════════════════┐

║  MoveIt                  Navigation(move_base)                遥控器/翻越规划 ║
└═══════════════════════════════════════════════════════════════════════════════┘
             │                          │                          │
             │FJT(6轴)                  │/cmd_vel                  │/flipper_controller/command
             ▼                          ▼                          ▼
      ┌──────────────────┐        ┌───────────────┐         ┌──────────────────┐
      │ arm_traj_splitter│        │diff_drive_ctrl│         │flipper_controller│

```

## 2. 下行数据流（命令：从上层到电机）

### 2.1 MoveIt 机械臂轨迹

```text
MoveIt
  └─ FollowJointTrajectoryGoal(6 joints)
      └─ arm_traj_splitter
          ├─ FJT(arm_joint_1~4) -> pp_arm_controller -> CanDriverHW.write() -> can_B -> 4台PP电机
          └─ FJT(arm_joint_5~6) -> canopen_arm_controller -> RobotLayer.write() -> can_A -> 2台CANopen电机
```

### 2.2 Navigation 底盘控制

```text
Navigation(move_base)
  └─ /cmd_vel (Twist)
      └─ diff_drive_controller
          └─ CanDriverHW.write() -> can_B -> 2台MT电机(left_track/right_track)
```

### 2.3 摆臂控制

```text
遥控器/自主模块
  └─ /flipper_controller/command (Float64MultiArray)
      └─ flipper_controller
          └─ RobotLayer.write() -> can_A -> 4台CANopen电机(flipper_1~4)
```

## 5. 控制频率与时序

```text
┌──────────────────────────────────────────────────────────────┐
│                        时间轴                                │
│                                                              │
│  t=0        MoveIt 发送 6 轴轨迹计划                          │
│  t=~0.5ms   splitter 收到，拆分                              │
│  t=~1ms     两个后端分别收到各自的子轨迹                       │
│             ┌──────────────────────────────────┐             │
│             │  100Hz 控制循环（每 10ms 一个周期）  │             │
│             │                                  │             │
│             │  can_driver_node:                 │             │
│             │    read() → pp_arm_ctrl 插值       │             │
│             │    → write() → CAN 帧             │             │
│             │                                  │             │
│             │  canopen_motor_node:              │             │
│             │    read() → canopen_arm_ctrl 插值  │             │
│             │    → write() → CAN 帧             │             │
│             └──────────────────────────────────┘             │
│  t=T        两个后端各自到达终点                               │
│  t=T+~1ms   splitter 合并结果返回 MoveIt                     │
└──────────────────────────────────────────────────────────────┘

两个后端开始执行的时间差：~0.5-2ms
在典型 3 秒运动周期中占比：< 0.1%
实验室场景完全可接受
```
