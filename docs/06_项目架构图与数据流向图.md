# 项目架构图与数据流向图

## 1. 总体架构图

```text
┌═══════════════════════════════════════════════════════════════════════════════┐
║ 上层应用                                                                      ║
║  MoveIt                  Navigation(move_base)                遥控器/翻越规划 ║
└═══════════════════════════════════════════════════════════════════════════════┘
             │                          │                          │
             │FJT(6轴)                  │/cmd_vel                  │/flipper_controller/command
             ▼                          ▼                          ▼
      ┌──────────────────┐        ┌───────────────┐         ┌──────────────────┐
      │ arm_traj_splitter│        │diff_drive_ctrl│         │flipper_controller│
      │ (C++ 代理节点)   │        │(can_driver内) │         │(canopen内)       │
      └───────┬──────────┘        └──────┬────────┘         └────────┬─────────┘
              │                           │                           │
      ┌───────┴────────┐                  │                           │
      ▼                ▼                  ▼                           ▼
pp_arm_controller   canopen_arm_controller   MT帧→can_B           CANopen PDO→can_A
(can_driver内,4轴)  (canopen内,2轴)

can_driver_node (can_B): arm_joint_1~4 + left/right_track
canopen_motor_node (can_A): arm_joint_5~6 + flipper_1~4
```

## 2. 下行数据流（命令：从上层到电机）

### 2.1 MoveIt 机械臂轨迹

```text
MoveIt
  └─ FollowJointTrajectoryGoal(6 joints)
      └─ arm_traj_splitter
          ├─ FJT(arm_joint_1~4) -> pp_arm_controller -> CanDriverHW.write() -> can_B -> 4台PP电机
          └─ FJT(arm_joint_5~6) -> canopen_arm_controller -> RobotLayer.write() -> can_A -> 2台CANopen电机
```

### 2.2 Navigation 底盘控制

```text
Navigation(move_base)
  └─ /cmd_vel (Twist)
      └─ diff_drive_controller
          └─ CanDriverHW.write() -> can_B -> 2台MT电机(left_track/right_track)
```

### 2.3 摆臂控制

```text
遥控器/自主模块
  └─ /flipper_controller/command (Float64MultiArray)
      └─ flipper_controller
          └─ RobotLayer.write() -> can_A -> 4台CANopen电机(flipper_1~4)
```

## 3. 上行数据流（状态：从电机到上层）

```text
can_B 电机反馈 -> can_driver_node -> /can_driver/joint_states
can_A 电机反馈 -> canopen_motor_node -> /canopen/joint_states

joint_state_publisher(source_list=[can_driver/joint_states, canopen/joint_states])
  -> /joint_states
  -> RViz / robot_state_publisher / MoveIt
```

## 4. 关键同步点

- 两条总线物理隔离：`can_A`(CANopen) 与 `can_B`(私有协议)
- 轨迹同步由 `arm_traj_splitter` 保证统一起始时序
- 上层统一只认 `/joint_states` 与标准控制接口
