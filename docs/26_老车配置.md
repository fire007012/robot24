# 老车配置

本文用于把当前老车的整车配置集中写在一处，便于现场联调、迁机和排障时快速核对。

本文以 2026-04-23 当前工作区现行配置为准。

---

## 1. 整车当前正式结构

当前老车按三层理解：

- 顶层启动编排：`robot_bringup`
- 统一硬件与生命周期门面：`Eyou_ROS1_Master/hybrid_motor_hw_node`
- 上层功能控制包：`mobility_control`、`flipper_control`、`arm_control`

当前正式执行入口：

- 顶层整机启动：`robot_bringup/launch/full_system.launch`
- 统一硬件门面：`Eyou_ROS1_Master/launch/hybrid_motor_hw.launch`

当前原则：

- 不再把 `car_control` 作为正式控制归属包，它只保留迁移期兼容能力。
- 不再把 `roscanopen/canopen_hw_node` 视为默认实机入口。
- 生命周期 authority 统一收口到 `hybrid_motor_hw_node`。

---

## 2. 三类电机与总线分配

当前老车包含三类执行对象：

### 2.1 底盘前轮 MT 电机

- 关节：`left_front_wheel_joint`、`right_front_wheel_joint`
- 协议：MT
- 后端：`can_driver`
- 总线：`can0`

当前基线理解：

- `can0` 承载底盘前两轮 MT 电机。
- `mobility_control/base_cmd_node` 下游接 `wheel_controller`，再进入 `can_driver`。

### 2.2 机械臂六轴 + 夹爪

- 机械臂组：`main_arm`
- 六轴关节：
  - `shoulder_yaw_joint`
  - `shoulder_pitch_joint`
  - `elbow_pitch_joint`
  - `wrist_pitch_joint`
  - `wrist_roll_joint`
  - `wrist_yaw_joint`
- 夹爪主动关节：`left_gripper_finger_joint`
- 夹爪从动关节：`right_gripper_finger_joint`，`mimic`

当前协议与总线：

- 前两轴 `shoulder_yaw_joint`、`shoulder_pitch_joint`：MT
- 后四轴与夹爪：PP
- 统一后端：`can_driver`
- 当前实机总线：`can1`

说明：

- 当前机械臂全轴已经归到 `can_driver`，不再作为默认 CANopen 链路。
- 历史上有 arm2 走 CANopen 的阶段，但当前默认不再这样使用。

### 2.3 摆臂四轴

- 归属包：`flipper_control`
- 当前后端：`hybrid -> can_driver -> ECB`
- 默认链路：`ecb://...`

当前理解：

- 摆臂四轴默认不占用 `can0/can1` SocketCAN。
- 它们通过 `can_driver` 的 ECB 网络设备接入。
- 若未来某个现场版本改成经 CAN 接入，再按 `can0` 归类；但当前标准配置不是这样。

## 3. 控制包与职责

### 3.1 robot_bringup

作用：

- 顶层整机启动编排
- 组织实机链路与 MoveIt 链路

不负责：

- 生命周期真相状态
- 直接执行底层电机命令

### 3.2 Eyou_ROS1_Master

作用：

- 统一 `can_driver` 与可选 CANopen 后端
- 对外提供统一生命周期服务
- 托管共享 `controller_manager`

当前正式生命周期服务：

- `init`
- `enable`
- `disable`
- `halt`
- `resume`
- `recover`
- `shutdown`

### 3.3 mobility_control

作用：

- 承接 `/cmd_vel`
- 速度限幅与超时刹停
- 输出到 `/wheel_controller/cmd_vel`

当前下游链路：

`/cmd_vel -> mobility_control/base_cmd_node -> /wheel_controller/cmd_vel -> wheel_controller -> can_driver -> MT 前轮`

### 3.4 flipper_control

作用：

- 摆臂目标管理
- 模式管理
- 安全约束

当前默认后端：

- `hybrid`
- 非默认 `canopen`

### 3.5 arm_control

作用：

- 机械臂目标执行
- 夹爪控制
- 与 MoveIt 执行链衔接

当前关键节点：

- `arm_goal_executor_node`
- `gripper_cmd_node`

## 4. 统一启动配置

### 4.1 顶层实机入口

当前整机实机默认入口：

```bash
roslaunch robot_bringup full_system.launch
```

当前含义：

- 默认通过 `Eyou_ROS1_Master` 拉起统一硬件门面。
- 默认控制路径为 `can_driver`。
- 默认不要求显式传入 CANopen DCF 和 joints 配置。

### 4.2 hybrid 默认基线

当前建议实机联调基线：

```bash
roslaunch Eyou_ROS1_Master hybrid_motor_hw.launch \
  auto_init:=false auto_enable:=false auto_release:=false
```

推荐原因：

- 启动后先保持人工确认。
- 便于现场排故。
- 避免上电即进入运动状态。

### 4.3 hybrid 当前关键参数

当前默认理解：

- `canopen_dcf_path`：默认留空
- `canopen_joints_path`：默认留空
- `can_driver_config`：默认来自 `can_driver/config/can_driver.yaml`
- `enable_ecb_control`：默认 `false`
- `optional_ecb_init`：默认 `true`
- `primary_can_device`：默认留空，自动探测
- `auto_detect_primary_can_device`：默认 `true`
- `auto_remap_single_socketcan_device`：默认 `true`
- `start_wheel_controller`：默认 `true`
- `start_gripper_controller`：默认 `false`

补充说明：

- `enable_ecb_control=false` 时，ECB 设备不参与初始化，但不会影响机械臂或底盘链路。
- 若需要让摆臂也参与统一生命周期，再显式打开 `enable_ecb_control=true`。

## 5. 控制器配置

当前整车主要控制器可按下面理解：

- `joint_state_controller`
- `arm_position_controller`
- `wheel_controller`
- `flipper_csp_controller`
- `flipper_csv_controller`
- `gripper_controller`

当前默认情况：

- `wheel_controller` 默认参与启动
- `gripper_controller` 默认不自动启动，需要时显式启动或通过 bringup 参数控制

## 6. 机械臂与 MoveIt 配置

当前 MoveIt 配置包：`car_moveit_config`

当前规划组：

- `main_arm`
- `gripper`

当前机械臂配置要点：

- 机械臂为 6 轴，不是历史 5 轴配置。
- 夹爪以单主动关节 `left_gripper_finger_joint` 为准。
- `group_name` 当前应使用 `main_arm`。

当前模型包：`car_urdf`

当前运行时 URDF：

- 底盘前轮关节
- 摆臂四轴关节
- 机械臂六轴关节
- 单主动夹爪 + mimic 从动夹爪

## 7. 生命周期使用约定

当前正式约定：

- 只有 `hybrid_motor_hw_node` 负责维护生命周期状态机。
- 上位机/HMI 可以发起生命周期操作，但不是状态机 owner。
- `arm_control`、`mobility_control`、`flipper_control` 不再各自维护独立生命周期。

当前推荐操作链：

- 初始化：`init`
- 进入运行：`enable -> resume`
- 停止到待机：`halt -> disable`
- 故障恢复：`recover`
- 关闭执行链：`shutdown`

---

## 8. 现场联调时的最小核对项

整车联调前，建议至少核对下面内容：

### 8.1 通讯层

- `can0` 是否在线
- `can1` 是否在线
- ECB 网段是否可达

### 8.2 电机分配

- 前轮 MT 是否在 `can0`
- 机械臂全轴是否在 `can1`
- 摆臂是否走 `ecb://...`

### 8.3 控制器

- `wheel_controller` 是否 running
- `arm_position_controller` 是否 running
- `flipper` 控制器是否按当前模式运行

### 8.4 生命周期

- `init` 是否成功
- `enable` 是否成功
- `resume` 后是否进入可运动状态

---

## 9. 当前老车配置结论

一句话总结当前老车：

- 底盘前轮 MT 走 `can0`
- 机械臂六轴 + 夹爪走 `can1`
- 摆臂四轴走 `ECB 网络设备`
- 统一执行与生命周期入口是 `Eyou_ROS1_Master/hybrid_motor_hw_node`
- 顶层整机入口是 `robot_bringup/full_system.launch`
- 默认实机链路以 `can_driver` 为主，CANopen 只保留为显式兼容后端

这就是当前老车的整车正式配置基线。
