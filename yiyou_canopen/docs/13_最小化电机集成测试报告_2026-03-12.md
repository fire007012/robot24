# yiyou_canopen 最小化电机集成测试报告（2026-03-12）

在 `yiyou_canopen` 包内进行最小化单电机集成测试，验证以下链路：
- ROS launch 能拉起 `canopen_motor_node`
- 主站与单节点 CANopen 设备（Node-ID=5）通信可用
- 驱动器可完成 CiA402 初始化并进入可控状态

## 4. 现象与排查过程（时间序列）

### 4.1 初始启动错误
现象：
- `Invalid <node> tag: respawn and required cannot both be set to true`

处理：
- 在 `yiyou_canopen_min.launch` 中新增参数 `canopen_respawn`，默认 `false`

### 4.2 缺少 robot_description / device
现象：
- `Could not find parameter robot_description on parameter server`
- `Device not set`

处理：
- 在最小化 launch 中加载 `robot_description`
- 将 `bus_single_5.yaml` 同时加载到 `canopen_motor_node` 私有命名空间

### 4.3 URDF 关节名不匹配
现象：
- `joint node_5 was not found in URDF`

处理：
- 新增 `canopen_single_5.urdf`，定义 `joint name="node_5"`

### 4.4 控制器起不来
现象：
- `Controller Spawner couldn't find the expected controller_manager ROS interface.`

解释：
- `canopen_motor_node` 未完成初始化时，不会拉起 controller_manager 接口

### 4.5 驱动初始化失败（核心问题）
现象：
- `rosservice call /canopen/driver/init` 返回：
  - `Transition timeout; Could not enable motor; Transition timeout`
- 日志伴随：
  - `RPDO timeout`

结论：
- 失败发生在 CiA402 使能阶段

## 5. 总线级实测结果（关键证据）

### 5.1 节点 ID 识别
上电抓包出现：
- `705 [1] 00`

结论：
- 节点 ID 为 `5`（`0x700 + 0x05`）

### 5.2 SDO 通信验证（手工帧）
发送：
- 读 `0x1000:0`：`605#4000100000000000`
- 读 `0x6041:0`：`605#4041600000000000`
- 读 `0x6061:0`：`605#4061600000000000`

回包：
- `585#4300100011082200`（设备类型可读）
- `585#4B41600050020000`（状态字曾返回 `0x0250`）
- `585#4F61600001000000`（模式显示 `0x01`）

结论：
- SDO 基础通信可用
- 当前模式读数显示为 `Profile Position`

### 5.3 手工 CiA402 控制字序列
发送：
- `6040=0x0006`（Shutdown）
- `6040=0x0007`（Switch on）
- `6040=0x000F`（Enable operation）

读回状态：
- `6041=0x0233`

附加验证：
- `603F=0x0000`（无故障码）

结论：
- 设备可进入 `Switched On`
- 不能进入 `Operation Enabled`
- `init` 超时与现场状态一致

## 6. 当前状态判断

### 6.1 已确认可用
- `can0` 接口可用（物理层 up）
- 节点 ID=5 确认无误
- SDO 可收发
- ROS 端参数路径已修正（`robot_description`、`bus/device`、`switching_state`）

### 6.2 未通过项
- `driver/init` 无法完成驱动使能
- controller_manager 无法创建
- 位置控制命令无法生效

### 6.3 根因定位（当前最可能）
非 ROS 配置主因，更偏向驱动器侧使能条件未满足：
- STO/硬件使能链路
- 急停回路
- 厂商伺服使能相关参数
- 运行条件互锁（即使 `603F=0`，也可能存在使能抑制条件）

## 7. 建议的后续动作
1. 在驱动器侧核查“允许运行”硬件条件：STO、Servo ON 输入、急停链。
2. 对照厂商手册核查进入 `Operation Enabled` 所需对象与顺序（含必须位）。
3. 先保留当前最小化配置做通信诊断；硬件使能链路确认后，将 `switching_state` 改回 `Operation_Enable` 再复测。
4. 复测通过标准：
- `/canopen/driver/init` 返回 `success: true`
- `/canopen/controller_manager/*` 服务可见
- `/canopen/joint_states` 连续发布
- `single_position_controller` 指令可触发实际运动
