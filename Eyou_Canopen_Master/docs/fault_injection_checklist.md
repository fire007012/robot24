# 故障注入验证清单

> 适用于 canopen_hw 驱动层的故障场景验证
> 日期：2026-03-19

---

## 使用说明

逐一执行以下故障场景，记录实际行为是否符合预期。每个场景执行前确保系统处于正常运行状态（所有轴 `OperationEnabled`）。

---

## 场景清单

### F01 — CAN 总线断线

| 项目 | 内容 |
|------|------|
| **注入方式** | 物理拔掉 CAN 线缆，或 `sudo ip link set can0 down` |
| **预期行为** | 所有轴触发 `heartbeat lost`，`is_operational` 置 false，日志输出 `[WARN] heartbeat lost` |
| **恢复方式** | 重新插入线缆 / `sudo ip link set can0 up`，等待心跳恢复 |
| **预期恢复** | 日志输出 `heartbeat recovered`，状态机从当前状态重新启动 |
| **验收** | [ ] 通过 / [ ] 不通过 |

---

### F02 — 单轴掉电 / 节点重启

| 项目 | 内容 |
|------|------|
| **注入方式** | 关闭某一轴伺服驱动器电源，等待 3 秒后重新上电 |
| **预期行为** | 该轴 `heartbeat lost` → `is_fault = true`，其余轴不受影响；`all_operational` 置 false |
| **恢复方式** | 驱动器重新上电，NMT boot 完成 |
| **预期恢复** | 该轴 `OnBoot` 触��，PDO 验证（如启用）完成后重新进入 `OperationEnabled` |
| **验收** | [ ] 通过 / [ ] 不通过 |

---

### F03 — PDO 映射不匹配

| 项目 | 内容 |
|------|------|
| **注入方式** | 修改 `master.dcf` 中某轴 RPDO/TPDO 映射参数，使其与驱动器实际映射不一致 |
| **预期行为** | `OnBoot` 中 PDO 验证失败，日志输出 `PDO mapping mismatch`，`pdo_verified_` 置 false，该轴 `is_operational` 保持 false |
| **恢复方式** | 修正 DCF 文件，重启节点 |
| **验收** | [ ] 通过 / [ ] 不通过 |

---

### F04 — PDO 验证 SDO 读取超时

| 项目 | 内容 |
|------|------|
| **注入方式** | 在 PDO 验证期间拔掉 CAN 线（`OnBoot` 成功后、SDO 读取中），或将超时设为极短值（1ms） |
| **预期行为** | `PdoMappingReader` 超时，日志输出 `PDO read failed: timeout`，`health_.pdo_verify_timeout` 计数 +1，该轴不进入 operational |
| **恢复方式** | 恢复 CAN 连接，触发 NMT 重启或手动重启节点 |
| **验收** | [ ] 通过 / [ ] 不通过 |

---

### F05 — 伺服 EMCY 紧急报文

| 项目 | 内容 |
|------|------|
| **注入方式** | 使电机堵转触发过流保护，或通过调试工具发送 EMCY 帧 |
| **预期行为** | 日志输出 `EMCY eec=0xNNNN er=0xNN`，`health_.emcy_count` 计数 +1，`last_emcy_eec` 记录错误码 |
| **恢复方式** | 消除堵转条件后，状态机自动执行故障复位流程 |
| **验收** | [ ] 通过 / [ ] 不通过 |

---

### F06 — 故障复位超过最大次数

| 项目 | 内容 |
|------|------|
| **注入方式** | 持续触发伺服故障（如堵转），使状态机连续执行 `max_fault_resets`（默认 3）次故障复位 |
| **预期行为** | 前 3 次自动复位尝试，第 4 次停止复位，状态机保持 `Fault` 状态，日志输出相应警告 |
| **恢复方式** | 人工介入消除故障根因后，需重启节点 |
| **验收** | [ ] 通过 / [ ] 不通过 |

---

### F07 — 心跳丢失后恢复

| 项目 | 内容 |
|------|------|
| **注入方式** | 短暂断开 CAN 线缆（< 2 秒）后重新连接 |
| **预期行为** | `heartbeat lost` → `heartbeat recovered`，`health_.heartbeat_lost` 和 `health_.heartbeat_recovered` 各 +1 |
| **恢复方式** | 自动恢复，无需人工介入 |
| **预期恢复** | 状态机继续正常运行，`is_operational` 恢复为 true |
| **验收** | [ ] 通过 / [ ] 不通过 |

---

### F08 — 有序关机（SIGINT）

| ��目 | 内容 |
|------|------|
| **注入方式** | 运行中发送 `kill -SIGINT <pid>` |
| **预期行为** | `GracefulShutdown()` 执行三阶段关机：DisableOperation → Shutdown → NMT Stop |
| **验收标准** | 退出码为 0，关机耗时 < 5 秒，所有轴进入 `ReadyToSwitchOn` 或更低状态 |
| **验收** | [ ] 通过 / [ ] 不通过 |

---

### F09 — 强制关机（SIGKILL）

| 项目 | 内容 |
|------|------|
| **注入方式** | 运行中发送 `kill -9 <pid>` |
| **预期行为** | 进程立即终止，无 graceful shutdown。伺服驱动器应依靠自身看门狗进入安全状态 |
| **关注点** | 确认无共享资源泄漏（CAN socket、文件描述符），重启后可正常运行 |
| **验收** | [ ] 通过 / [ ] 不通过 |

---

### F10 — OnBoot 配置失败重试

| 项目 | 内容 |
|------|------|
| **注入方式** | 在驱动器 boot 过程中短暂干扰 CAN 通信，使 `OnBoot` 收到 `es != 0` |
| **预期行为** | 自动发送 `RESET_NODE` 重试，最多 `max_boot_retries`（默认 3）次，`health_.boot_retries` 计数递增 |
| **恢复方式** | 如重试成功则自动恢复；如重试耗尽则标记 PDO 验证失败 |
| **验收** | [ ] 通过 / [ ] 不通过 |
